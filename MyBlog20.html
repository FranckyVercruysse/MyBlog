Enabling EF Core migrations

In this section, we start using the EF Core Code First Migrations feature for managing data model changes.
Code First Migrations automates the process of making incremental changes to a data model and propagating those changes to the database.

The EF tools for the command-line interface (CLI) are provided in Microsoft.EntityFrameworkCore.Tools.DotNet. To install this package, add it to the DotNetCliToolReference collection in the .csproj file, as shown. You can edit the .csproj file by right-clicking the project name in Solution Explorer and selecting Edit MyBlog.csproj 
<Project Sdk="Microsoft.NET.Sdk.Web">

  <PropertyGroup>
    <TargetFramework>netcoreapp2.0</TargetFramework>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.AspNetCore.All" Version="2.0.0" />
  </ItemGroup>

  <ItemGroup>
    <DotNetCliToolReference Include="Microsoft.EntityFrameworkCore.Tools.DotNet" Version="2.0.0" />
    <DotNetCliToolReference Include="Microsoft.VisualStudio.Web.CodeGeneration.Tools" Version="2.0.0" />
  </ItemGroup>

</Project>

When you executed the migrations add command, EF generated the code that will create the database from scratch. This code is in the Migrations folder, in the file named <timestamp>_InitialCreate.cs (20171103015843_InitialCreate.cs).
The Up method of the InitialCreate class creates the database tables that correspond to the data model entity sets, and the Down method deletes them.

Apply the migration to the database

In the command window, enter the following command to create the database and tables in it.
dotnet ef database update



The output from the command is similar to the migrations add command, except that you see logs for the SQL commands that set up the database.

You can use SQL Server Object Explorer to inspect the database and you'll notice the addition of an __EFMigrationsHistory table that keeps track of which migrations have been applied to the database.
View the data in that table and you'll see one row for the first migration.
